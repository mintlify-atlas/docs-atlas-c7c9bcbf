## Overview

The `PaymentsResource` provides methods to fetch information about your account's payment assets, including coin balances and subscription status.

Accessible via `client.payments` (sync) or `client.payments` (async).

## Methods

### assets()

Fetch the current account's payment assets and subscription status, including coin balances and active subscription flags.

<CodeGroup>
```python Sync
from seaart import SyncClient

with SyncClient() as client:
    client.auth.login(email="user@example.com", password="password")
    
    # Get payment assets
    info = client.payments.assets()

        print("Has unlimited queue")
```

## Usage Examples

### Checking Coin Balance

```python
from seaart import SyncClient

with SyncClient() as client:
    client.auth.login(email="user@example.com", password="password")
    
    # Get current balance
    info = client.payments.assets()
    
    total_coins = info.value.calculate_coins
    
    if total_coins > 0:
        print(f"You have {total_coins} coins available")
        print(f"  - Paid: {info.value.paid_coins}")
        print(f"  - Free: {info.value.free_coins}")
        print(f"  - Temp: {info.value.temp_free_coins}")
    else:
        print("No coins available. Please recharge or get free coins.")
```

### Checking Subscription Status

```python
from seaart import SyncClient
from datetime import datetime

with SyncClient() as client:
    client.auth.login(email="user@example.com", password="password")
    
    info = client.payments.assets()
    
    if info.value.is_active:
        # Format expiry time
        expiry = datetime.fromtimestamp(info.value.expiry_time)
        
        print(f"Active subscription: {info.value.vip_type}")
        print(f"Expires: {expiry.strftime('%Y-%m-%d %H:%M:%S')}")
        
        # Check tier
        if info.value.big_is_active:
            print("Pro/Big tier active")
        elif info.value.lite_is_active:
            print("Lite tier active")
    else:
        print("No active subscription")
        print("Consider upgrading for more features!")
```

### Pre-Generation Check

```python
from seaart import SyncClient

def can_generate(client: SyncClient) -> bool:
    """Check if the account can generate images."""
    info = client.payments.assets()
    
    # Check if coins are available
    if info.value.calculate_coins <= 0:
        print("Insufficient coins")
        return False
    
    print(f"Available coins: {info.value.calculate_coins}")
    return True

with SyncClient() as client:
    client.auth.login(email="user@example.com", password="password")
    
    if can_generate(client):
        task = client.images.txt2img(
            prompt="a beautiful sunset",
            model_no="model_abc123"
        )
        result = client.tasks.wait(task.value.id)
        print(f"Generated: {result.image_url}")
        
        # Check balance after generation
        new_info = client.payments.assets()
        print(f"Remaining coins: {new_info.value.calculate_coins}")
```

## Best Practices

<Tip>
  **Check balance before generation**: Call `assets()` before starting expensive operations to ensure sufficient coins are available.
</Tip>

<Note>
  **Understand coin types**: The account uses different coin types (paid, free, temp). The `calculate_coins` field provides the total usable amount.
</Note>

<Warning>
  **Monitor limits**: Free accounts have lower `run_limit` and `hang_limit` values. Upgrade to a subscription for higher limits and additional features.
</Warning>
